## Breve explicação
- **S**: Single responsibility principle
  - Determina que cada classe/arquivo da nossa aplicação deve ter apenas uma unica responsabilidade, isso significa que ele deve ser responsavel por executar uma unica tarefa dentro da aplicação.

- **O**: Ope/closed principle
  - Define que uma classe deve ser aberta para extensões, mas não para modificações.

- **L**: Liskov substitution principle
  - As classes derivadas devem ser substituíveis por suas classes bases.

- **I**: Interface segregation principle
  - Criar interfaces granulares a específicas.


- **D**: Dependency inversion principle
  - Dependa de abstrações e não de implementações.
